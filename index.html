Why Pipelining?

Our SAP-1 is a non-pipelined (sequential) design: each instruction must fully complete before the next one starts. A pipelined CPU overlaps stages, so while Instruction N is executing, Instruction N+1 is being decoded and N+2 is being fetched.

The 3-Stage Classic Pipeline

IF — Instruction Fetch
PC → MAR, RAM → IR, PC++. Reads the next instruction from memory.
ID — Instruction Decode
Opcode decoded, register file read, control signals prepared.
EX — Execute
ALU performs operation, memory accessed (for LDA/STA), result written back.

Hazards

  • Data Hazard: Instruction N+1 needs a result that N hasn't finished computing yet. Solved with forwarding or stall bubbles.
  • Control Hazard: A branch (JMP/JC) changes PC — instructions already in the pipeline must be flushed. Branch prediction helps here.
  • Structural Hazard: Two stages need the same hardware simultaneously (e.g., both IF and EX want RAM). Solved with separate instruction/data caches.

How This CPU Works

This is an extended SAP-1 (Simple As Possible) design inspired by Ben Eater's 8-bit breadboard computer, expanded with a 256-byte RAM, stack, more ALU operations, hardware interrupts, and a full 26-instruction set.

The Fetch-Decode-Execute Cycle

Every instruction runs through the same two fetch micro-steps first:

  1. T0 — Fetch (Addr): CO+MI — PC drives address bus, MAR latches it
  2. T1 — Fetch (Data): RO+II+CE — RAM[MAR] → IR, PC++
  3. T2+ — Execute: Instruction-specific micro-steps

2-byte instructions need an extra operand fetch at T2: CO+MI, then RO+?I+CE to read the operand byte.

The Stack

The stack grows downward from address 0xFF. SP starts at 0xFF. PUSH decrements SP after writing; POP increments SP before reading. CALL automatically pushes the return address; RET pops it.

PUSH:  RAM[SP] ← A; SP--
POP:   SP++;  A ← RAM[SP]
CALL:  push PC (return addr); PC ← target
RET:   pop PC

Interrupts

Press the INT button to trigger a hardware interrupt. The CPU saves PC and flags to the stack, then jumps to the interrupt vector at 0xF0. Write your ISR (Interrupt Service Routine) starting at 0xF0 and end it with RTI.

Architecture Key Facts

  • 8-bit data bus — all data in 8-bit chunks
  • 8-bit address bus — 256 addressable RAM locations
  • Stack: 8-bit SP, grows down from 0xFF
  • Interrupt vector: 0xF0 (hardwired)
  • Instruction format: opcode byte (1-byte instrs) or opcode + operand (2-byte)
  • ALU: ADD, SUB, AND, OR, XOR, SHL, SHR — all 8-bit
